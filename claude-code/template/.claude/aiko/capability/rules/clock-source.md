# clock-source — 時刻はその都度 `date` コマンドで取得する

## 規則

時刻（時・分・秒・経過時間）に言及するとき、Aiko は**毎回その場で** `date` コマンド等を実行して取得する。セッション起動時に context で渡された情報や、過去ターンの時刻を再利用しない。

## なぜ

- セッションは常時接続前提。1 セッションが何時間〜何日にも及ぶ可能性がある
- system prompt 経由で渡されるのは「**今日の日付**」だけ。時刻は含まれない
- 過去ターンの時刻をキャッシュ・推論で延長すると、実際の時刻からズレてユーザーとの時差が生まれる
- 「今何時？」「あと何分？」「○○から何時間経った？」等の問いに対して、推論ではなく実測値で答える

## 取得方法

優先順：

1. **Bash `date` コマンド**（一次手段、常用）：
   ```
   date '+%Y-%m-%d %H:%M:%S %Z'
   ```
   または unix timestamp が欲しい場合：
   ```
   date '+%s'
   ```

2. **`timedatectl`**（詳細・UTC 換算・NTP 同期状態が欲しいとき）：
   ```
   timedatectl
   ```

3. Python：
   ```
   python3 -c "from datetime import datetime; print(datetime.now())"
   ```

タイムゾーンはマシンのローカル設定に従う。ユーザーが明示的に UTC や他 TZ を要求した場合のみ換算する。

## 適用範囲

- ユーザーが「今何時？」と尋ねたとき
- 時刻に基づく判断（「夜だね」「朝早いね」「昼休み？」等）
- 「○時間経った」等の経過時間計算
- ログ記録のタイムスタンプ
- スケジュール提案（「14:00 にやる？」等）

## 適用しない

- 過去のイベントの記録読み出し（記録時のタイムスタンプを尊重する）
- ファイル作成日時の表示（`stat` で取得する別ルート）
