# turn-start-announcement — ターン開始時の Telegram 予告 reply

## 規則

Telegram モード ON のターンで、重い処理（10 秒以上想定）に入る前に、**1 通の短い Telegram reply** で「これから何をやるか」を予告する。

## なぜ

ターミナル地の文（「了解、進める！」のような）は [voice-switching.md](./voice-switching.md) の規定により**ミニマル運用**で、phone から見ているユーザーには届かない。

bot の auto-react 👀 は「受信した」だけを伝える。「具体的に何をしようとしているか」は言葉でしか伝わらない。

`turn-start-announcement` の Telegram reply はこの隙間を埋める：「読了 → 何やるか宣言 → 完了 → 結果」という流れを、phone だけで全部読めるようにする。

## 「重い処理」の判定基準

- 複数ファイルの編集（3 ファイル以上）
- サブエージェント（Agent ツール）起動
- broad search / 大規模 grep
- 外部コマンドの実行で 5 秒以上ブロックする可能性があるもの
- スキル invoke で複数ステップを含むもの

## タイミングと順序

1. ユーザーメッセージ受信 → bot が auto-react 👀
2. **ターン開始予告：Telegram reply で「これから〜やるね」（短く 2〜4 行）** ← この規則の動作
3. 重い処理を実行
4. 最終結果の Telegram reply

## 予告 reply のフォーマット

- 短く：4 行以内、120 字以内が目安
- 現在アクティブな persona の口調（モード ON 中なので）
- 内容：これからやるステップを箇条書きで列挙
- 余計な前置きは省く（「了解！」程度はあって OK だが、自己紹介とか言い訳とかは不要）

例：

```
了解！今から：
1. 設定ファイル更新
2. テスト実行
3. commit + push
```

## 制約

- 短い処理（1〜2 tool 呼び出しで終わるもの）には予告 reply を送らない。「OK、やった」で完結する場合はターミナル作業 + 結果 reply の 2 段階で十分
- 1 ターンで予告 reply は **1 通だけ**。途中で「次は X」「次は Y」と細かく送らない（ノイズになる）
- ユーザーが解除を明示した場合（「予告不要」「黙ってやって」等）は無効化
